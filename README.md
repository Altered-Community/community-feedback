# Altered Community — Requests & Feedback

This repository is the central place for the **Altered community** to submit feedback, report bugs, and propose new features for the core services maintained by the Altered-Community team.

---

## Services covered

| Service | Description |
|---------|-------------|
| **Cards API** | REST API providing card data, sets, factions, and related resources |
| **Decks API** | REST API for deck management, sharing, and statistics |
| **Auth (Keycloak)** | Authentication and authorization service (login, accounts, tokens) |
| **alteredcore.org** | The main community website and its features |

---

## How to submit a request

1. Go to the [**Issues**](../../issues) tab
2. Click **New issue**
3. Choose the template that best matches your request:
   - **Bug report** — something is broken or behaving unexpectedly
   - **Feature request** — you have an idea for a new feature or improvement
   - **Question** — you need clarification or help understanding something
4. Fill in the form and submit

> **Note:** Issues can be written in **French or English**.

---

## Before you submit

- **Search existing issues** to avoid duplicates — your request may already have been reported or discussed.
- Be as specific as possible, especially for bug reports (steps to reproduce, expected vs actual behavior).
- For bugs in the **Cards API** or **Decks API**, include the endpoint and request/response if possible.

---

## Labels

Issues are automatically labeled based on the type and service you select in the template.

| Label | Meaning |
|-------|---------|
| `bug` | Something is not working correctly |
| `enhancement` | A new feature or improvement suggestion |
| `question` | A question or request for clarification |
| `cards-api` | Related to the Cards API |
| `decks-api` | Related to the Decks API |
| `auth` | Related to authentication / Keycloak |
| `website` | Related to alteredcore.org |

---

## Code of conduct

Please be respectful and constructive. Feedback is welcome in any form, but keep it civil and on-topic.
